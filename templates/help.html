How to Write a Problem¶

Overview¶

The user interfaces in QBank for each problem type consists of text boxes and buttons. The text boxes also accepts HTML and JavaScript.

The What's this? button gives helpful indicators on the purpose of different text boxes and what different parameters can be added to author problems for a particular problem type. It also tells you some functions that can be used to make more effective questions.

Problem¶

Problem Name¶

A unique identifier that is used to : 1. Store a problem in the database. 2. Refer to a problem in different exported formats. 3. Refer to while creating a summative problem written from previously authored problems.

Difficulty¶

It just classifies problems as easy , medium or hard. This information can be used by an Intelligent Tutoring System (ITS), which bases the difficulty of the next question on the assessment of the previous one.

If correct, a question with a higher difficulty is posed or vice-versa.

Variables¶

This allows for generation of different problem instances based on a static Problem Template with variables that take on different specified values.

Variables are typically defined by using a <var>...</var> element. The content of a <var>...</var> block will be executed as JavaScript, with access to to all the properties and methods provided by the JavaScript Math object, as well as all the properties and methods defined in the modules which you have included.

  • Variable Name: ID for the variable that can be used to refer to it.

  • Variable Value: Values that the variable can take are specified here. This can be commma separated values or functions that are accepted by a publishing tool/ parsed into a compatible format.

For example to make a variable named YEAR that is a number from 2000 to 2013 you would do:

<!-- Random year from 2000 to 2013. -->
Variable Name  : YEAR
Variable Value : 2000+rand(13) or randRange(2000,2013)

For a list type of problem, use comma separated values enclosed in double-quotes:

<!-- Numbers from 1-5 -->
Variable Name  : A
Variable Value : "1", "2", "3", "4", "5"

Most mathematical problems that you generate will have some bit of randomness to them (in order to make for an interesting, not-identical, problem). You can generate these values up-front so that you can re-use them again in your problems.

Variable Reference¶

Inside of a <var>...</var> you can refer to other variables that you’ve already defined (order matters) by simply referencing their name.

For a list type of dynamic problem, refer to the variable using the variable name and a randomizer in square brackets. For instance, the first variable declared will have a randomizer "x1" and the second "x2" and so forth. 

<!-- Division -->
Variable Name  : A
Variable Value : "101", "222", "763", "41", "25"
Question       : What is the quotient of <var>A[x1]</var> / 12?

Another example where we define two variables (AVG and TIME ) and then multiply them together and store them in a third variable ( DIST ).

<!-- Compute one value based upon two others. -->
<var id="AVG">31 + rand(9)</var>
<var id="TIME">1 + rand(9)</var>
<var id="DIST">AVG * TIME</var>

Solution¶

The solution consists of the answer.

Answer¶

The answer can be any of the following.

  1. A valid choice
  2. A function that is the calculation of a question ( with specified values for variables)
  3. A previously defined variable.
Example
Answer : <var>round(DIST)</var>
The solution is automatically calculated depending on the values the variables take on runtime.

Another example for a static multiple choice answer.

Answer : None of the above.
The solution also gets added as one of the choices in the problem and doesn't explicitly have to be mentioned.

Choices¶

This can include text which acts as distractors for the user. The choices can also use the previosly defined variables.

Hints¶

These are textual suggestions to help the user figure out the correct answer. The hints can also use the previously authored variable .

The hints are optional.

Scripts¶

The author can add scripts written in JavaScript to add different functionality to the question (e.g. Interactivity). This option is provided for the Tool Specific type of problem.

QBank supports authoring questions in the Khan Academy Exercise format. For more information on writing Khan Academy exercises, go to this link.

Common Introduction¶

This is useful since the problems combined together can have some information that isn’t explicitly part of the statement of the question.
For example, formulae in a Physics or Math problem and values from the periodic table in Chemistry can be included here.

Available functions and Tips¶

Generating Random Numbers¶

  1. randRange( min, max ) - Get a random integer in [min, max].
  2. randRange( min, max, count ) - Get a random integer between min and max, inclusive. If count is specified, will return an array of random integers in the range.
  3. randRangeUnique( min, max, count ) - Get an array of unique random numbers between min and max, inclusive.
  4. randRangeExclude( min, max, excludes ) - Get a random integer between min and max, inclusive, that is never any of the values in the excludes array.
  5. randRangeNonZero( min, max ) - Get a random integer between min and max that is never zero.
  6. randFromArray( arr ) - Get a random member of arr.

Problem Type Specifics¶

Parameterized Question – List¶

Author problems with variables that take values that are explicitly specified.

User Interface components:

  1. Variables: The values for the variables need to specified in double-quotes separated by commas.

  2. Generate Answer Key: It shows the various combination of the different values the variable can take.

    This also provides an easy interface for the author to write answers corresponding to the variable combination. On clicking, "Generate Solution", the answers get added in the solution box.

  3. Solution:

    The answer takes a one dimensional array, with comma separated values in double-quotes. Use the "Generate Answer Key" button to write and generate the solution automatically.

Parameterized Question – Range¶

This type of question is generally suited for math problems which involves calculations.

User interface components:

  1. Variables: The values the variables take are between a range of values that are specified using randRange(min,max) function.

  2. Solution: The answer is the exact calculation within <var> .. </var> delimiters. The author doesn’t need to explicitly make the calculation and write the correct answer. This ensures the validity of answer and eliminates the risk of a mistake.

    Question: Calculate <var>A</var> + <var>B</var> - <var>C</var>
Solution: <var>A + B - C</var>

Another important feature of this type of problem is, the author doesn’t need to explicitly provide choices, since the user is expected to fill the answer in the blank provided.

Multi-Part Question¶

It is comprised of different previously authored questions from the Repository. You can combine different type of questions, free-form and multiple-choice simple questions and matching questions in your multipart question.

Common Introduction
The problems share a common introduction which generally contains information that is common to the questions.
Problem Name
Specify the exact identifier for a Problem. You can browse the existing problems by clicking the "Existing problems" button. You can then click "Add" and the problem gets added.

Tool Specific Question – Khan Academy Exercise¶

The user interface components for this type of problem is consistent with QBank but it supports inputs that specified by Khan Academy Exercise format.

Check out Khan Academy Exercise Framework Documentation.

Also, the What's this button gives a lot of direction in assisting a problem while authoring a question.
